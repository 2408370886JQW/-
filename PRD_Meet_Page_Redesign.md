# 产品需求文档 (PRD)：相见页重构 - “这次见面怎么安排”

## 1. 文档概述
| 项目 | 内容 |
| :--- | :--- |
| **文档名称** | 相见页重构 PRD |
| **版本号** | v1.0 |
| **状态** | 待开发 |
| **核心目标** | 将“相见页”从“团购列表”转型为“场景化见面流程策划工具” |

---

## 2. 产品定位与核心价值

### 2.1 当前问题 (AS-IS)
*   **现状**：相见页 = 团购页。
*   **用户感知**：“我进来是买便宜券的”。
*   **问题**：偏离社交定位，沦为同质化的本地生活工具，缺乏“人”的连接感。

### 2.2 目标定位 (TO-BE)
*   **新定位**：相见页 = **“这次见面怎么安排”**。
*   **用户感知**：“我进来是安排一次完美的见面”。
*   **核心价值**：
    1.  **卖关系**：针对不同社交关系（情侣、闺蜜、兄弟）提供专属方案。
    2.  **卖场景**：针对不同见面目的（初次约会、好久不见、庆祝）提供场景化建议。
    3.  **卖流程**：提供一站式活动流（吃 -> 玩 -> 住），而非单一店铺推荐。
    4.  **工具化**：团购券仅作为流程中的支付工具，而非平台核心。

---

## 3. 功能结构设计

### 3.1 页面层级结构
相见页将分为三个核心层级：

#### 第一层：场景选择层 (Entry Level)
*   **核心逻辑**：我和谁见？为什么见？
*   **UI 呈现**：顶部水平滚动或网格布局的场景标签/卡片。
*   **分类示例**：
    *   **关系维度**：💑 约会、👯 闺蜜、🧑‍🤝‍🧑 兄弟、👨‍👩‍👧‍👦 家庭
    *   **目的维度**：🎂 纪念日、🌙 深夜微醺、📸 拍照打卡、💬 坐坐聊聊、🍽 第一次约会

#### 第二层：流程方案层 (Solution Level)
*   **核心逻辑**：基于选定的场景，推荐完整的活动流程（Plan），而非单一商家。
*   **UI 呈现**：垂直流式布局的“流程卡片”。
*   **卡片内容示例**：
    *   **标题**：💡 第一次约会标准流程
    *   **流程概览**：🍽 吃饭（适合聊天） -> 🎬 看电影（拉近距离） -> ☕️ 咖啡（意犹未尽）
    *   **标签**：#不尴尬 #氛围感 #高成功率

#### 第三层：流程详情与执行层 (Execution Level)
*   **核心逻辑**：用户点击某个流程卡片后，进入具体的执行页面。
*   **UI 呈现**：分步骤的详情页。
*   **结构**：
    *   **Step 1: 吃饭**
        *   推荐理由：环境安静，适合初次交流。
        *   **推荐商家列表**（此处才出现具体的店和团购信息）：
            *   商家 A（评分 4.8，¥128/人）
            *   商家 B（评分 4.6，¥98/人）
    *   **Step 2: 看电影**
        *   推荐理由：缓解初次见面的紧张感。
        *   **推荐地点**：附近影院列表。
    *   **Step 3: 后续**
        *   推荐：附近的清吧或甜品店。

### 3.2 新增交互功能：发布/创建
*   **入口**：底部导航栏中间的“+”号（或页面内的“定制”按钮）。
*   **功能**：【创建一次约会 / 聚会】
*   **交互流程**：
    1.  点击“+”。
    2.  弹出表单/对话框：
        *   **跟谁？**（选择：情侣 / 朋友 / 哥们）
        *   **目的？**（选择：约会 / 打卡 / 聊天）
        *   **想要几步？**（多选：吃饭 / 看电影 / 住宿 / 娱乐）
    3.  系统生成：根据输入，自动生成一套推荐流程。

---

## 4. UI/UX 设计要求 (Design Guidelines)

### 4.1 视觉风格
*   **去电商化**：减少大红大绿的促销标签，使用更具质感的场景图片。
*   **情感化**：使用Emoji、插画或暖色调图标来表达社交关系。
*   **卡片式设计**：流程方案采用大卡片设计，突出“方案感”。

### 4.2 交互细节
*   **标签切换**：顶部的场景标签支持快速切换，下方内容即时刷新。
*   **流程预览**：在第二层卡片上，用户应能直观看到流程的步骤（Step 1 -> Step 2 -> Step 3）。

## 5. 开发计划 (Phase 28)

1.  **数据结构定义**：
    *   定义 `Scenario` (场景/关系)
    *   定义 `Plan` (流程方案)
    *   定义 `Step` (流程步骤)
    *   关联 `Merchant` (商家) 到 `Step`
2.  **组件开发**：
    *   `ScenarioSelector`: 顶部场景选择器
    *   `PlanCard`: 流程方案展示卡片
    *   `PlanDetail`: 流程详情页（模态框或新页面）
3.  **页面重构**：
    *   修改 `Home.tsx` 中的 `activeTab === "meet"` 渲染逻辑。
    *   替换原有的 `GROUP_BUY_DEALS` 列表。

---

## 6. 总结
通过本次重构，我们将彻底改变“相见页”的基因，使其从一个**“货架”**转变为一个**“社交参谋”**，真正解决用户“见面干什么”的痛点，从而提升产品的社交粘性和差异化竞争力。
